# Grafana Runtime Federation OS

This repository defines Grafana as a Runtime Governance and Collapse Control view for the mobility operating business.

## Top rule

Grafana is not a KPI dashboard, BI screen, infrastructure monitor, or IoT list.

Grafana is the runtime view for identifying where mobility operation collapses.

## Structure

1. Federation Runtime
2. Responsibility Runtime
3. Collapse Runtime
4. Runtime Governance
5. Company Runtime View
6. Monetization Mapping

## Operation rule

Apps may change freely.
Grafana does not directly generate or overwrite app UI.
Grafana audits runtime consistency, responsibility boundaries, synchronization, and collapse propagation.

The current phase allows many collapse states because most apps are not yet connected.
Development progress is measured by collapse reduction, not by the number of UI features.

## Company view modes

### Toyota

Positioning: integrated collapse control for autonomous mobility operation.

Main value: designing the runtime conditions that make autonomous mobility viable as an operating business.

### Denso

Positioning: translation from OEM runtime collapse to control code, runtime modules, SDKs, and supply proposals.

Main value: selling control and code from the viewpoint of OEM operation collapse, not from parts alone.

### Honda

Positioning: energy and autonomous mobility runtime.

Main value: connecting energy business, PPA, VPP, Scope3, node operation, and mobile operation.

Honda is a separate business axis and must not be mixed into Toyota-group positioning.
